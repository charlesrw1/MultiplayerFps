## Lua-backed Component reflection (WIP, uncommitted)
- New `PROP_LUA_BACKED` flag on `PropertyInfo`: storage lives in a per-instance shadow buffer on the owning `ClassBase`, not at `inst + offset`. `PropertyInfo::get_ptr` branches on the flag; all editors/serializer paths route through it.
- `Component` now carries `unique_ptr<uint8_t[]> lua_field_shadow` + cached `lua_owner_type`. `lua_class_alloc` allocates and seeds the shadow from the template Lua table; `~Component` runs `destruct_lua_field` over non-POD fields and unregisters from `LuaClassTypeInfo::live_instances`.
- `LuaClassTypeInfo` synthesizes a `PropertyInfoList` from `---@type` annotations on Component subclasses. Supported scalars: `number/float`, `integer/int`, `boolean/bool`, `string`, enum-by-name. Untyped table entries are dropped from reflection (still accessible from Lua at runtime).
- Hot-reload merge runs in 4 phases: snapshot live instances against old layout → tear down old shadows → rebuild metadata → reallocate shadows seeded from new template + restore overlapping snapshot values.
- Editor wiring: Lua component classes appear in the add-component picker; `EdPropertyGrid::invalidate_cache` is fired via `ScriptManager::on_class_reloaded` so cached `const PropertyInfo*` rows don't dangle past a reload.
- Unit tests in `scripting_lua_component_reflection_test.cpp` cover parser + layout synthesis. Integration tests in `test_lua_component.cpp` cover place/edit/save/reload, hot-reload migration, grid rows, hot-reload-while-selected crash regression, destructor/unregister, multi-instance reload, dropped-field save warning, and two-class no-alias.
